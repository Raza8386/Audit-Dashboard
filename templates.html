1. Purpose and Scope

1.1 Purpose

This policy establishes guidelines for the acceptable use of Artificial Intelligence (AI) technologies within {{ORGANIZATION_NAME}}. The policy aims to:

  • Enable productive and innovative use of AI tools while managing associated risks
  • Protect sensitive data, intellectual property, and confidential information
  • Ensure compliance with applicable laws, regulations, and ethical standards
  • Promote responsible AI use that aligns with organizational values and risk tolerance
  • Establish clear accountability for AI system deployment and usage

1.2 Scope

This policy applies to:

  • Personnel: All employees, contractors, consultants, temporary staff, and third parties working on behalf of {{ORGANIZATION_NAME}}
  • AI Systems: Generative AI tools, LLMs and chatbots, ML models, AI-powered analytics, RPA with AI capabilities, computer vision and NLP systems
  • Activities: Development, procurement, deployment, integration, and day-to-day use of AI systems
  • Data: Any organizational or customer data used with AI systems

2. Definitions

TermDefinition
Artificial Intelligence (AI)Systems demonstrating intelligent behavior by analyzing their environment to achieve specific goals. Includes ML, deep learning, NLP, and computer vision.
Generative AIAI capable of generating new content (text, images, audio, video, code). Examples: ChatGPT, Claude, Midjourney, GitHub Copilot.
Large Language Model (LLM)AI models trained on vast text data to understand and generate human-like text. A subset of Generative AI.
Approved AI ToolsAI systems that have completed {{ORGANIZATION_NAME}}'s approval process. See: {{APPROVED_AI_TOOLS_LIST_LOCATION}}.
Sensitive DataInformation requiring protection: PII, PHI, financial data, trade secrets, and Confidential/Restricted data under the Data Classification Policy.

3. Governance Structure

3.1 AI Governance Committee

{{ORGANIZATION_NAME}} maintains a {{AI_GOVERNANCE_COMMITTEE}} responsible for approving AI tools, reviewing risk assessments, establishing risk tolerance, monitoring incidents, and recommending policy updates.

Composition: Representatives from Information Security, Legal, Compliance, Privacy, IT, Business Units, and Executive Leadership. Meeting Cadence: Quarterly, with ad-hoc meetings for urgent approvals or incident response.

3.3 Escalation Paths

  • Standard AI Tool Request: Employee → Manager → AI Governance Committee
  • Policy Violation Report: Reporter → Manager → AI Governance Committee → Legal/HR
  • High-Risk AI Decision: Business Unit → AI Governance Committee → Executive Leadership

4. Approved AI Tools

4.1 Approval Process

  1. Check Approved Tools List at {{APPROVED_AI_TOOLS_LIST_LOCATION}}
  2. Submit Request (if tool not approved) — include tool name, vendor, use case, data classification, business justification
  3. Risk Assessment — Information Security and Privacy assess data security, vendor posture, data residency, terms of service
  4. Committee Review — AI Governance Committee approves, denies, or approves with conditions
  5. Procurement — IT/Procurement negotiates contract with required security terms
  6. Communication — Approved tool added to tools list with approved use cases

Approval Timeline: Standard 15 business days; expedited 5 business days.

4.3 Prohibited AI Tools

  • Free/consumer versions of AI tools (e.g., free ChatGPT, free Bard) — use only enterprise versions
  • Tools from vendors lacking adequate security certifications or required contractual terms
  • Tools that train on customer data without opt-out
  • Tools hosted in prohibited geographic regions
  • Shadow AI tools procured without IT/Security approval

5. Acceptable Use Guidelines

5.1 Permitted Uses

  • Content Creation: Drafting documents, editing, brainstorming, translation (non-sensitive)
  • Code Development: Generating snippets, debugging, documenting code (approved tools only)
  • Data Analysis: Analyzing non-sensitive datasets, generating reports, summarizing information
  • Research and Learning: Researching topics, explaining complex concepts, learning new skills
  • Process Automation: Automating repetitive tasks, classifying documents, extracting information

5.2 Prohibited Uses

  • Sensitive Data Processing: Inputting PII, PHI, or trade secrets into non-approved tools
  • Automated Decision-Making Without Review: Employment, loan, credit, medical decisions solely from AI
  • Harmful Uses: Deepfakes, harassment content, security circumvention, malware creation
  • Misrepresentation: Presenting AI content as human-created where disclosure is required
  • IP Violations: Infringing copyrights, patents, trademarks; plagiarizing works

5.3 Output Review Requirements

All AI-generated content must be: (1) reviewed by a human, (2) validated for accuracy, (3) checked for bias, (4) assessed for appropriateness, and (5) edited as needed.

Personnel are accountable for content they use or distribute, regardless of whether it was AI-generated.

1. Purpose and Scope

This framework establishes a systematic approach for identifying, assessing, mitigating, and monitoring risks associated with AI systems at {{ORGANIZATION_NAME}}. It aligns with NIST AI RMF 1.0 (GOVERN/MAP/MEASURE/MANAGE), ISO/IEC 42001:2023 (Clauses 6.1, 8.1, 9.1), and EU AI Act Article 9.

Risk Categories Covered

  • Bias and fairness risks
  • Privacy and data protection risks
  • Security and adversarial risks
  • Safety and reliability risks
  • Transparency and explainability limitations
  • Compliance and legal risks
  • Ethical and reputational risks

4. Risk Identification (MAP)

4.1 AI Risk Categories

4.1.1 Bias and Fairness Risks

AI system produces discriminatory outcomes or systematically disadvantages protected groups.

Risk Factors: Biased/unrepresentative training data; features correlating with protected characteristics; model amplifying societal bias; no fairness testing; deployment context differs from training context.

Potential Impacts: Discriminatory decisions (hiring, lending, insurance); anti-discrimination law violations; regulatory penalties; reputational damage.

4.1.2 Privacy and Data Protection Risks

AI system violates privacy rights or fails to protect personal data.

Risk Factors: Processing PII/PHI; inadequate anonymization; unclear third-party vendor data usage; training without consent; re-identification risk.

Potential Impacts: GDPR, CCPA, HIPAA violations; regulatory fines; inability to honor data subject rights.

4.1.3 Security and Adversarial Risks

AI system is vulnerable to attacks or can be exploited by adversaries.

Key threats: Model poisoning, adversarial examples, model extraction, backdoors, prompt injection (LLMs), insufficient access controls.

4.1.4 Safety and Reliability Risks

AI system failures or inaccuracies cause harm or unsafe conditions.

Risk Factors: Safety-critical deployment; performance degradation over time (data/concept drift); insufficient edge-case testing; no fail-safe mechanisms.

4.1.5 Transparency and Explainability Risks

Inability to explain AI decisions or understand how the system reaches conclusions.

Risk Factors: Black-box models (deep neural networks); no explanation mechanism; regulatory explainability requirements (GDPR Article 22, fair lending laws).

4.1.6 Compliance and Legal Risks

Risk Factors: Applicable regulations not identified; IP concerns with training data; lack of required assessments (DPIA, FRIA).

4.1.7 Ethical and Reputational Risks

Risk Factors: Use case conflicting with ethical norms; potential harm to vulnerable populations; controversial contexts (surveillance, scoring, manipulation).

  4. How many people are affected?
  5. In what geographic regions is the AI deployed?

  2. Have you tested the model for fairness across demographic groups?

  5. Is there a process for individuals to challenge AI decisions?

3. Stage 1: Problem Definition and Scoping

Key Activities

  1. Business Case Development: Define problem and objectives; identify stakeholders; estimate ROI; assess ML alternatives.
  2. Feasibility Assessment: Confirm data availability; evaluate technical feasibility; identify skill gaps.
  3. Ethical Review: Identify ethical concerns (bias, privacy, manipulation); assess impact on vulnerable populations.
  4. Success Criteria Definition: Define measurable metrics (accuracy, F1, AUC); establish fairness criteria; set acceptable thresholds.

Gate 1: Approval to Proceed

Approvers: Business Sponsor, {{STANDARD_OWNER}} / Data Science Lead, Ethics Reviewer, and for high-risk models: AI Governance Committee.

Documented in: {{APPROVAL_WORKFLOW_TOOL}}

4. Stage 2: Data Acquisition and Preparation

Key Activities

  1. Data Acquisition: Identify sources; verify permissions and legal rights; document data lineage.
  2. Data Governance Review: Classify data; check sharing agreements for ML restrictions.
  3. Privacy Assessment: Identify PII/PHI; complete DPIA if required; apply data minimization; document legal basis (GDPR Article 6).
  4. Bias Assessment: Analyze training data representativeness; check for historical bias.
  5. Data Preparation: Clean data; feature engineering; split into training/validation/test sets; document all preprocessing for reproducibility.

Required Documentation

  • Dataset Card (Datasheets for Datasets format) — motivation, composition, collection, preprocessing, uses, distribution, maintenance
  • Privacy Impact Assessment (if PII/PHI involved)
  • Bias Assessment Report — known biases, underrepresented groups, mitigation strategies

5. Stage 3: Model Development

Key Activities

  1. Algorithm Selection: Evaluate candidate algorithms; justify selection on problem type, interpretability, performance, computational constraints.
  2. Model Training: Train on training set; tune hyperparameters; implement cross-validation; track experiments in {{ML_PLATFORM}}.
  3. Fairness Testing: Test for disparate impact; calculate demographic parity, equal opportunity, equalized odds, predictive parity; apply bias mitigation if needed.
  4. Interpretability: For high-stakes decisions: implement LIME, SHAP, or attention mechanisms; validate with domain experts.
  5. Version Control: Commit code to Git; version training data; register model in {{MODEL_REGISTRY}}.

Required Documentation

  • Model Card — model details, intended use, factors, metrics, evaluation data, ethical considerations, caveats
  • Algorithm Justification
  • Fairness Testing Report

6. Stage 4: Validation and Testing

Testing Checklist

  • Performance Validation: Evaluate on held-out test set; compare to baseline and success criteria; test edge cases and OOD data.
  • Fairness Validation: Re-test fairness metrics; confirm no disparate impact on protected groups.
  • Robustness Testing: Test with noisy/corrupted inputs; evaluate sensitivity to perturbations; validate confidence calibration.
  • Security Testing: Test adversarial inputs (FGSM, PGD); check for data leakage; for LLMs: test prompt injection and jailbreaks.
  • UAT: Validate with end users in realistic scenarios; collect feedback on outputs and explanations.
  • Risk Assessment: Complete AI Risk Assessment (per the AI Risk Assessment Framework).

Gate 4: Deployment Approval

Approvers: Data Science Lead, Business Sponsor, Information Security, and for high-risk models: AI Governance Committee, Legal/Compliance.

7. Stage 5: Deployment

Key Activities

  1. Deployment Planning: Define architecture (batch/real-time/edge); plan rollout strategy (canary, blue-green, phased); prepare rollback plan.
  2. Monitoring Setup: Configure performance monitoring in {{MONITORING_PLATFORM}}; set up drift detection; implement alerts for accuracy drop, fairness violations, high error rates, adversarial inputs.
  3. Human-in-the-Loop Controls: Implement human review for edge cases or low-confidence predictions; define confidence thresholds.
  4. Operational Handoff: Train operations team; provide runbook; document escalation procedures; establish on-call rotation.

8. Stage 6: Monitoring and Maintenance

Drift Detection

  • Data Drift: Changes in input feature distributions — use KS test, Chi-square test; alert when distributions diverge.
  • Concept Drift: Changes in relationship between features and outcome — monitor ground truth vs. predictions.
  • Prediction Drift: Changes in model output distribution.

Retraining Triggers

  • Performance degrades below threshold (e.g., 5% accuracy drop)
  • Significant data drift detected
  • 6 months of new training data accumulated
  • Business requirements change
  • Security vulnerability discovered

Review Cadence

Model Risk LevelReview Frequency
High RiskQuarterly

9. Stage 7: Retirement

Retirement Triggers

  • Model superseded by a new version or alternative approach
  • Business use case no longer exists
  • Model performance unacceptable and cannot be improved
  • Regulatory changes make model non-compliant
  • Security vulnerabilities cannot be adequately mitigated
  • Maintenance cost exceeds business value

Required Documentation

  • Retirement Plan: Timeline, stakeholder communication, transition plan
  • Decommissioning Checklist: Confirmation all production systems updated
  • Data Retention Record: What was retained, what was deleted, retention periods
  • Archival Index: Where documentation is archived
  • Lessons Learned Report: Retrospective on model lifecycle
Archive minimum 7 years for high-risk models (or per regulatory requirements).

#!/usr/bin/env python3
"""
AWS SOC 2 Evidence Collector
Collects compliance evidence from AWS for SOC 2 audit preparation.
"""

import boto3
import json
import csv
import argparse
import os
from datetime import datetime
from typing import Dict, List, Any


class AWSEvidenceCollector:
    """Collects SOC 2 compliance evidence from AWS services."""

    def __init__(self, profile: str = None, region: str = None):
        session_args = {}
        if profile:
            session_args['profile_name'] = profile
        session_args['region_name'] = region or 'us-east-1'
        try:
            self.session = boto3.Session(**session_args)
            self.region = self.session.region_name
            sts = self.session.client('sts')
            self.account_id = sts.get_caller_identity()['Account']
        except Exception as e:
            raise RuntimeError(f"Failed to initialize AWS session: {e}")
        self.timestamp = datetime.utcnow().isoformat() + 'Z'

    def collect_iam_evidence(self) -> Dict[str, Any]:
        """CC6.1, CC6.2, CC6.3 — IAM user access controls."""
        print("Collecting IAM evidence...")
        iam = self.session.client('iam')
        evidence = {
            'collection_timestamp': self.timestamp,
            'account_id': self.account_id,
            'control_mapping': ['CC6.1', 'CC6.2', 'CC6.3'],
            'users': []
        }
        paginator = iam.get_paginator('list_users')
        for page in paginator.paginate():
            for user in page['Users']:
                name = user['UserName']
                mfa   = iam.list_mfa_devices(UserName=name)['MFADevices']
                keys  = iam.list_access_keys(UserName=name)['AccessKeyMetadata']
                groups = iam.list_groups_for_user(UserName=name)['Groups']
                policies = iam.list_attached_user_policies(UserName=name)['AttachedPolicies']
                evidence['users'].append({
                    'username': name,
                    'mfa_enabled': len(mfa) > 0,
                    'access_keys': [{'key_id': k['AccessKeyId'], 'status': k['Status']} for k in keys],
                    'groups': [g['GroupName'] for g in groups],
                    'policies': [p['PolicyName'] for p in policies],
                })
        evidence['summary'] = {
            'total_users': len(evidence['users']),
            'users_with_mfa': sum(1 for u in evidence['users'] if u['mfa_enabled']),
        }
        return evidence

    def collect_encryption_evidence(self) -> Dict[str, Any]:
        """CC6.7 — Encryption at rest for S3, RDS, EBS."""
        print("Collecting encryption evidence...")
        evidence = {'s3_buckets': [], 'rds_instances': [], 'ebs_volumes': []}
        s3 = self.session.client('s3')
        for b in s3.list_buckets()['Buckets']:
            try:
                enc = s3.get_bucket_encryption(Bucket=b['Name'])
                algo = enc['ServerSideEncryptionConfiguration']['Rules'][0]['ApplyServerSideEncryptionByDefault']['SSEAlgorithm']
                evidence['s3_buckets'].append({'name': b['Name'], 'encrypted': True, 'algorithm': algo})
            except:
                evidence['s3_buckets'].append({'name': b['Name'], 'encrypted': False})
        return evidence

    def collect_cloudtrail_evidence(self) -> Dict[str, Any]:
        """CC7.2 — CloudTrail logging configuration."""
        print("Collecting CloudTrail evidence...")
        ct = self.session.client('cloudtrail')
        trails = []
        for t in ct.describe_trails()['trailList']:
            status = ct.get_trail_status(Name=t['Name'])
            trails.append({
                'name': t['Name'],
                'multi_region': t.get('IsMultiRegionTrail', False),
                'is_logging': status.get('IsLogging', False),
                'log_validation': t.get('LogFileValidationEnabled', False),
            })
        return {'trails': trails, 'summary': {'total': len(trails)}}

    def export_json(self, evidence, output_dir):
        os.makedirs(output_dir, exist_ok=True)
        path = os.path.join(output_dir, f'aws_evidence_{self.timestamp.replace(":", "-")}.json')
        with open(path, 'w') as f:
            json.dump(evidence, f, indent=2, default=str)
        print(f"Exported: {path}")


def main():
    parser = argparse.ArgumentParser(description='AWS SOC 2 Evidence Collector')
    parser.add_argument('--profile')
    parser.add_argument('--region', default='us-east-1')
    parser.add_argument('--output', default='./evidence')
    parser.add_argument('--format', choices=['json', 'csv', 'both'], default='json')
    args = parser.parse_args()

    collector = AWSEvidenceCollector(profile=args.profile, region=args.region)
    evidence = {
        'iam':        collector.collect_iam_evidence(),
        'encryption': collector.collect_encryption_evidence(),
        'logging':    collector.collect_cloudtrail_evidence(),
    }
    if args.format in ['json', 'both']:
        collector.export_json(evidence, args.output)
    print("Evidence collection complete!")


if __name__ == '__main__':
    exit(main())

#!/usr/bin/env bash
# GitHub SOC 2 Configuration Checker
# Requirements: gh CLI (authenticated), jq

set -euo pipefail
VERSION="1.0.0"
RED='\033[0;31m'; GREEN='\033[0;32m'; YELLOW='\033[1;33m'; BLUE='\033[0;34m'; NC='\033[0m'
ORG=""; REPO=""; OUTPUT_FORMAT="text"
PASS_COUNT=0; FAIL_COUNT=0; WARN_COUNT=0; SKIP_COUNT=0

log_pass() { echo -e "${GREEN}[PASS]${NC} $1"; ((PASS_COUNT++)); }
log_fail() { echo -e "${RED}[FAIL]${NC} $1"; [[ -n "${2:-}" ]] && echo "       Remediation: $2"; ((FAIL_COUNT++)); }
log_warn() { echo -e "${YELLOW}[WARN]${NC} $1"; ((WARN_COUNT++)); }
log_skip() { echo -e "${BLUE}[SKIP]${NC} $1"; ((SKIP_COUNT++)); }

check_mfa_requirement() {
  local mfa_required
  mfa_required=$(gh api "/orgs/${ORG}" --jq '.two_factor_requirement_enabled' 2>/dev/null || echo "false")
  if [[ "$mfa_required" == "true" ]]; then
    log_pass "CC6.1: MFA required for organization members"
  else
    log_fail "CC6.1: MFA not required for organization members" \
             "Enable at Settings > Authentication security > Require two-factor authentication"
  fi
}

check_branch_protection() {
  local repo="$1"
  local default_branch
  default_branch=$(gh api "/repos/${ORG}/${repo}" --jq '.default_branch' 2>/dev/null)
  local protection
  protection=$(gh api "/repos/${ORG}/${repo}/branches/${default_branch}/protection" 2>/dev/null || echo "{}")
  if [[ "$protection" == "{}" ]]; then
    log_fail "CC8.1: No branch protection on ${repo}:${default_branch}" \
             "Enable at Settings > Branches > Add branch protection rule"
    return
  fi
  local require_pr
  require_pr=$(echo "$protection" | jq -r '.required_pull_request_reviews // empty')
  if [[ -n "$require_pr" ]]; then
    log_pass "CC8.1: Branch protection configured on ${repo}:${default_branch}"
  else
    log_warn "CC8.1: Branch protection incomplete on ${repo}:${default_branch}"
  fi
}

check_dependabot() {
  local repo="$1"
  local enabled
  enabled=$(gh api -X GET "/repos/${ORG}/${repo}/vulnerability-alerts" 2>/dev/null && echo "true" || echo "false")
  if [[ "$enabled" == "true" ]]; then
    log_pass "CC3.2: Dependabot alerts enabled on ${repo}"
  else
    log_fail "CC3.2: Dependabot alerts not enabled on ${repo}" \
             "Enable at Settings > Security & analysis > Dependabot alerts"
  fi
}

check_secret_scanning() {
  local repo="$1"
  local status push
  status=$(gh api "/repos/${ORG}/${repo}" --jq '.security_and_analysis.secret_scanning.status // "disabled"')
  push=$(gh api "/repos/${ORG}/${repo}" --jq '.security_and_analysis.secret_scanning_push_protection.status // "disabled"')
  if [[ "$status" == "enabled" && "$push" == "enabled" ]]; then
    log_pass "CC6.1: Secret scanning + push protection enabled on ${repo}"
  elif [[ "$status" == "enabled" ]]; then
    log_warn "CC6.1: Secret scanning on but push protection off on ${repo}"
  else
    log_fail "CC6.1: Secret scanning not enabled on ${repo}" \
             "Enable at Settings > Security & analysis > Secret scanning"
  fi
}

main() {
  while [[ $# -gt 0 ]]; do
    case $1 in
      --org)    ORG="$2";    shift 2 ;;
      --repo)   REPO="$2";   shift 2 ;;
      --format) OUTPUT_FORMAT="$2"; shift 2 ;;
      *)        shift ;;
    esac
  done
  [[ -z "$ORG" ]] && { echo "Error: --org required"; exit 1; }
  check_mfa_requirement
  local repos=()
  if [[ -n "$REPO" ]]; then
    repos=("$REPO")
  else
    mapfile -t repos < <(gh repo list "$ORG" --json name --jq '.[].name' --limit 1000)
  fi
  for repo in "${repos[@]}"; do
    check_branch_protection "$repo"
    check_dependabot        "$repo"
    check_secret_scanning   "$repo"
  done
  local total=$((PASS_COUNT + FAIL_COUNT + WARN_COUNT))
  local rate=$((total > 0 ? PASS_COUNT * 100 / total : 0))
  echo "Pass Rate: ${rate}%"
  [[ $FAIL_COUNT -eq 0 ]] && echo "All checks passed!" || echo "${FAIL_COUNT} check(s) failed"
}

main "$@"

#!/usr/bin/env bash
# Policy Template Generator
# Replaces {{VARIABLE}} placeholders from a YAML file or interactively.
# Prerequisites: yq (mikefarah version)

set -euo pipefail
RED='\033[0;31m'; GREEN='\033[0;32m'; YELLOW='\033[1;33m'; NC='\033[0m'
TEMPLATE=""; VARS_FILE=""; OUTPUT=""; INTERACTIVE=false; KEEP_FRONTMATTER=false

log_info()  { echo -e "${GREEN}[INFO]${NC} $1"; }
log_warn()  { echo -e "${YELLOW}[WARN]${NC} $1"; }
log_error() { echo -e "${RED}[ERROR]${NC} $1"; }

extract_variables() {
  grep -oE '\{\{[A-Z_0-9]+\}\}' "$1" 2>/dev/null | sort -u | sed 's/[{}]//g' || true
}

replace_variables() {
  local template="$1" vars_file="$2"
  local content; content=$(cat "$template")
  while IFS= read -r var; do
    [[ -z "$var" ]] && continue
    local value; value=$(yq eval ".${var} // \"\"" "$vars_file" 2>/dev/null || echo "")
    if [[ -n "$value" ]]; then
      content="${content//\{\{${var}\}\}/${value}}"
      log_info "Replaced {{${var}}}"
    else
      log_warn "No value for {{${var}}}"
    fi
  done < <(extract_variables "$template")
  echo "$content"
}

interactive_replace() {
  local template="$1"
  local content; content=$(cat "$template")
  while IFS= read -r var; do
    [[ -z "$var" ]] && continue
    local desc example required
    desc=$(yq eval ".variables[] | select(.name == \"$var\") | .description // \"\"" "$template" 2>/dev/null || echo "")
    example=$(yq eval ".variables[] | select(.name == \"$var\") | .example // \"\"" "$template" 2>/dev/null || echo "")
    required=$(yq eval ".variables[] | select(.name == \"$var\") | .required // false" "$template" 2>/dev/null || echo "false")
    echo "Variable: ${var}"
    [[ -n "$desc"    ]] && echo "  Description: ${desc}"
    [[ "$required" == "true" ]] && echo "  Required: YES"
    local value
    read -rp "  Value [${example}]: " value
    value="${value:-$example}"
    [[ "$required" == "true" && -z "$value" ]] && { log_error "Required: ${var}"; exit 1; }
    content="${content//\{\{${var}\}\}/${value}}"
    echo ""
  done < <(extract_variables "$template")
  echo "$content"
}

strip_frontmatter() {
  sed '1 { /^---/ { :a N; /\n---/! ba; d} }'
}

validate_output() {
  local remaining
  remaining=$(grep -oE '\{\{[A-Z_0-9]+\}\}' "$1" 2>/dev/null | sort -u || true)
  [[ -n "$remaining" ]] && log_warn "Unreplaced variables remain: $remaining"
}

main() {
  while [[ $# -gt 0 ]]; do
    case $1 in
      --template)        TEMPLATE="$2";    shift 2 ;;
      --vars)            VARS_FILE="$2";   shift 2 ;;
      --output)          OUTPUT="$2";     shift 2 ;;
      --interactive)     INTERACTIVE=true; shift ;;
      --keep-frontmatter) KEEP_FRONTMATTER=true; shift ;;
      *) shift ;;
    esac
  done
  [[ -z "$TEMPLATE" || -z "$OUTPUT" ]] && { log_error "--template and --output required"; exit 1; }
  local result
  if [[ "$INTERACTIVE" == "true" ]]; then
    result=$(interactive_replace "$TEMPLATE")
  else
    [[ -z "$VARS_FILE" ]] && { log_error "--vars required in non-interactive mode"; exit 1; }
    result=$(replace_variables "$TEMPLATE" "$VARS_FILE")
  fi
  [[ "$KEEP_FRONTMATTER" == "false" ]] && result=$(echo "$result" | strip_frontmatter)
  echo "$result" > "$OUTPUT"
  log_info "Generated: $OUTPUT"
  validate_output "$OUTPUT"
}

main "$@"
